フラッディングとは	スイッチが、MACアドレステーブルに宛先MACが載っていないとき、しかたなく全ポートから送信すること。	network
IGMP	Internet Group Management Protocol<br>マルチキャストグループへの参加や離脱をホストが通知したり、マルチキャストグループに参加しているホストの有無をルータがチェックするときに使うプロトコル	network
IGMPスヌーピング	スイッチが、IGMPメッセージの情報をもとに、マルチキャストフレームを必要なポートだけに転送させること。	network
TCP<ul><li>フルネーム</li><li>特徴</li></ul>	Transmission Control Protocol<ul><li>コネクション型通信。3ウェイハンドシェイクで通信相手とコネクションを確立。</li><li>フロー制御を行う。再送制御、順序制御</li><li>低速</li></ul>	network
TCPとUDPを使ったプロトコル	<ul><li>TCP:<br>HTTP, SMTP, POP3, SSH, FTP, Telnet, DNS(ゾーン転送)</li><li>UDP:<br>DNS(問合せ), SNMP, DHCP, TFTP, NTP</li></ul>	network
TCPヘッダの最初の4項目	送信元port (16bit), 宛先port (16bit), シーケンス番号(32bit), 確認応答番号(32bit)	network
3ウェイハンドシェイクの流れ	SYN (コネクション要求元 -> 要求先)<br>SYN+ACK (要求元 <- 要求先)<br>ACK (要求元 -> 要求先)	network
TCPのコネクション終了の流れ	FIN (要求元 -> 要求先)<br>ACK (要求元 <- 要求先)<br>FIN (要求元 <- 要求先)<br>ACK (要求元 -> 要求先)	network
TCPで、確認応答を待たずにデータを送信できる仕組みは何か	スライディングウィンドウ<br>ウィンドウサイズ内であれば確認応答を待たずにデータを送信できる。	network
UDP<ul><li>フルネーム</li><li>特徴</li></ul>	User Datagram Protocol<ul><li>コネクションレス型</li><li>高速</li></ul>	network
UDPヘッダ	送信元port (16bit), 宛先port (16bit), セグメント長(16bit), チェックサム(16bit)	network
ポート番号の範囲	0~65535<br>(0~1023: well-known ポート)	network
静的ルーティング、動的ルーティングのメリット、デメリット	[静的ルーティング]<br>メリット: 設計者の意図通りの確実な経路選択がされる。<br>デメリット: 大規模だと経路情報が大量になり間違いが起こりやすい。<br>[動的ルーティング]<br>メリット: 自動で最適な経路選択。障害時に自動で経路変更可能。<br>デメリット: ルーティング情報交換用にわずかだがトラフィックが流れる。ルータに負荷がかかり処理が遅くなることがある。	network
