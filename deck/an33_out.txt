PKI	Public Key Infrastructure: 公開鍵基盤<br>公開鍵を利用したセキュリティの基盤	network
ディジタル署名とは	文書の元データのハッシュ値を作成者の秘密鍵で暗号化したもの。元データの付加して送る。	network
ディジタル署名の目的	<ul><li>改ざん防止（完全性）</li><li>なりすまし防止（真正性）</li><li>否認防止</li></ul>書類そのものは暗号化されない	network
受信側の、ディジタル署名の検証の仕組み	<ol><li>受け取ったデータをハッシュ</li><li>受け取ったディジタル署名を検証鍵（作成者の公開鍵）で復号</li><li>上記1,2が一致するか確認</li></ol>	network
ディジタル証明書とは何を証明するものか	公開鍵	network
証明書の発行や失効を行う機関は	認証局<br>CA: Certification Authority<br>	network
認証局の種類	<ul><li>公的な認証局（ベリサインなど）</li><li>プライベート認証局（企業などが独自で構築）</li></ul>	network
ディジタル証明書において、証明書にディジタル署名をする際の、署名鍵と検証鍵はそれぞれ何か。	<ul><li>署名鍵: CAの秘密鍵</li><li>検証鍵: CAの公開鍵</li></ul>	network
証明書は3種類。ひとつはサーバ証明書。あとふたつは何か。	ルート証明書（CAの公開鍵を証明）、クライアント証明書（クライアントの公開鍵を証明）<br>サーバ証明書はサーバの公開鍵を証明。	network
CRL	Certificate Revocation List: 証明書の失効のリスト。	network
