RIPとは	Routing Information Protocol<br>ディスタンスベクタ型のルーティングプロトコル。<br>ベクトルの大きさはホップ数(通過するルータの数)、方向はネクストホップ(次のルータ)	network
RIPのデメリット(2点)	<ul><li>経路情報伝達が一定時間間隔 -> 障害発生時、う回路を見つけるのに時間がかかる。</li><li>ネットワークの回線速度を考慮できない。</li></ul>	network
RIPに対して、RIP2の改良点	<ul><li>情報交換をマルチキャストに改良 (RIPはUDPでブロードキャスト)</li><li>サブネットマスクに対応</li><li>更新情報ごとにパスワード認証</li><li>(最大ホップ数は同じ: 15)</li></ul>	network
RIPng	IPv6に対応したRIP。ng = next generation	network
OSPF<ul><li>フルネーム</li><li>特徴 3点</li></ul>	Open Shortest Path First<br>特徴<ul><li>リンクステータス型のアルゴリズム<br>宛先までの回線コストの総和が最小のパスを経路として選択(回線コストは回線速度に反比例)</li><li>エリアを分割する</li><li>LSA(Link State Advertisement)と呼ばれる情報を交換</li></ul>	network
OSPFでコストが同じ場合はどういう動作をするか	負荷分散される	network
OSPFでエリア分割する理由。形態。	ネットワークが大規模になるとルータにかかる負荷が増えて転送速度や切替り速度が遅くなるなど問題が発生するのでその対策。<br>エリア間をバックボーンで結ぶ形態を採る。	network
LSAの種類	Link State Advertisement<ul><li>Router-LSA: (各ルータから) 基本</li><li>Network-LSA: (DRから) エリア内の情報。DRを中心として交換</li><li>summary-LSA: (ABRから) エリア間の情報</li><li>AS-External-LSA: (ASBRから) AS間の情報</li></ul>	network
OSPFのルータの種類	<ul><li>代表ルータ (DR: Designated Router)<br>エリア内に一つずつ選出されたルータ。<br>(BDR: バックアップ)</li><li>エリア境界ルータ (ABR: Area Border Router)<br>エリアの境界上に位置するルータ</li><li>AS境界ルータ (ASBR: AS Border Router)<br>別のASとの接続点となるルータ</li></ul>	network
ASとは<br>AS番号について	Autonomous System (自律システム)<br>単一のルーティングポリシにより管理されるネットワーク。ほぼほぼ各ISPや企業。<br>AS番号はIANAが割り当てる。16ビット、0~65535	network
IGPとEGP<ul><li>フルネーム</li><li>意味</li><li>代表的なルーティングプロトコル</li></ul>	<ul><li>IGP: Internal Gateway Protocol<br>AS内で利用<br>RIP, OSPF</li><li>EGP: External Gateway Brotocol<br>AS間で利用<br>BGP</li></ul>	network
