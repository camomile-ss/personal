UTMとは。	Unified Threat Management<br>FW機器, IDS/IPS機器, SPAM対策機器などの機能を一つの筐体に統合。	network
UTMの機能	<ul><li>FW機能</li><li>VPN機能</li><li>侵入防止機能</li><li>バッファオーバフロー攻撃遮断機能</li></ul>ほかにアンチスパム機能, URLフィルタリング機能をもつものがある	network
HA	High Availability<br>FWの冗長化の仕組み。<br>冗長化プロトコルは(1)VRRPを利用したものと(2)独自プロトコルを利用したものがある。<br>VRRPではセッション情報などの同期ができないため(2)がほとんど。	network
EAPはどこが規定したか	IETF(Internet Engineering Task Force)	network
PPPの拡張EAPができた理由	PPPは1対1の通信。最近は無線LANなどオープンなネットワークが増え、自由に参加しやすいネットワークであり、セキュリティに関し高い認証が求められるようになってきた。	network
IEEE802.1X認証を実現するのに必要なもの三つ。	<ol><li>サプリカント<br>supplicant<br>認証要求をお願いするソフトウェア。クライアントPCにインストールされる。</li><li>オーセンティケータ<br>authenticator<br>認証Switch, 無線APなど</li><li>認証サーバ<br>RADIUSサーバに代表される</li></ol>	network
IEEE802.1X認証の手続き	PCのサプリカントからオーセンティケータに、さらに認証サーバに認証要求が行われ、逆向きに認証許可が行われる。	network
認証SWから認証サーバへ問い合わせる代表的なプロトコル	RADIUS。（以前はダイヤルアップのアクセスサーバなどで使われていたプロトコル）	network
IEEE802.1 PCをネットワークに接続してから認証が許可されるまでの流れ	<ol><li>PCにネットワークケーブルを接続する、または、PCの電源を入れる</li><li>スイッチから認証要求（サプリカントによりID入力画面が起動）</li><li>利用者がID, PW入力</li><li>認証SWは認証サーバにID, PWが正しいか確認</li><li>認証許可</li><li>スイッチングHUBにてportの状態を認証済みに</li><li>PCのインタフェースが有効になり通信可能に。</li></ol>	network
IEEE802.1X で、認証が成功していない状態でなぜ認証の要求フレームが届くのか。	認証開始時の接続要求フレーム（EAPOLフレーム）はMACアドレスが 01-80-C2-00-00-03。このフレームは特別に通信が許可される。	network
